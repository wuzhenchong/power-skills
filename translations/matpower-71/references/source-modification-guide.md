# 源码修改指南

当任务需要改变 MATPOWER 源码行为时使用本指南。

## 不可突破的边界

不要原地修改用户电脑上安装的 MATPOWER 包。

应把相关 MATPOWER `.m` 文件复制到当前项目或工作区中，只修改这些项目本地副本。目标位置是灵活的：可以是项目根目录、`src/`、overrides 目录、vendored 目录，或项目已有的 MATLAB 源码目录。关键是安装包保持不变。

## 编辑前确认

1. 原安装包中的 MATPOWER 源文件。
2. 将要编辑的项目本地副本。
3. 为什么普通扩展机制不够用。
4. 受影响的调用链。
5. 修改后要运行的测试或 case。

路径优先级检查：

```matlab
which runpf
which opf_setup
```

如果意图覆盖某个函数，`which` 结果必须指向项目本地副本。

## 优先使用扩展点

复制源码前，先判断是否可以通过以下方式完成：

- 修改 case 数据。
- 设置 `mpoption`。
- 修改 `gencost`。
- 使用 OPF 扩展字段。
- 添加 `userfcn` callback。

只有需求位于 MATPOWER 算法或包装逻辑内部，且无法通过支持的扩展点表达时，才应复制源码。

## 编辑习惯

- 本地副本尽量接近 MATPOWER 7.1 原文件。
- 在每处有意本地修改附近添加简短注释。
- 修改数值逻辑时避免大规模重构。
- 除非项目控制所有调用方，否则不要改函数签名。
- 时刻注意内部编号和外部编号。
- 先跑小 case，再跑用户目标 case。
