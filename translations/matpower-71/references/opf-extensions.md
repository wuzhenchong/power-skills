# OPF 扩展

自定义 OPF 目标函数或约束时，应选择侵入性最低的机制。

## 决策顺序

1. 如果修改标准 case 数据即可完成，就修改 `gencost`、限值、负荷、支路容量或电压上下限。
2. 如需额外线性/二次成本和约束，使用 MATPOWER OPF 扩展字段。
3. 如需分阶段转换数据、添加模型项、打印或保存自定义字段，使用 `userfcn` callback。
4. 只有以上机制无法表达需求时，才修改项目本地的 MATPOWER 源码副本。

## OPF 可识别扩展字段

- `A`、`l`、`u`：线性约束。
- `H`、`Cw`、`N`、`fparm`：广义成本。
- `z0`、`zl`、`zu`：用户变量初值和上下界。

复杂建模前应查本地或官方 `opf` help。

## Callback 扩展

- `ext2int`：把自定义输入数据转换到内部编号。
- `formulation`：向 OPF 模型添加变量、约束或成本。
- `int2ext`：把自定义结果转换回外部编号。
- `printpf`：添加自定义输出。
- `savecase`：保存自定义字段。
